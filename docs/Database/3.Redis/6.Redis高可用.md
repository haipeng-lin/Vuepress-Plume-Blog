## 2.哨兵机制

### 2.1 为什么要有哨兵机制

在 Redis 版本 2.8 开始之后，都提供了哨兵机制，目的是实现主从节点故障转移。

它会监测主节点是否存活，如果发现主节点挂了，它就会选举一个从节点切换为主节点，并且把新主节点的相关信息通知给从节点和客户端

### 2.2 哨兵机制的工作过程

哨兵其实一个特殊的 Redis 进程，观察者节点。

哨兵主要有三项工作内容：**监控、选主、通知**
