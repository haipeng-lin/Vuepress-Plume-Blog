## 一、前言

​		Spring AI 是 Spring 生态中专为 AI 工程设计的应用框架，简化了与大型语言模型 （LLM）、嵌入模型和向量数据库的集成，让企业级 Java 应用轻松接入 AI 能力，如聊天机器人、RAG（Retrieval Augmented Generation）和智能代理。

优点：

- Java 原生：无需切换到 Python，保持端到端 Java 开发
- Spring 集成：无缝嵌入 Spring Boot，支持自动装配和 Starter
- 多提供商支持：统一 API 接入 OpenAI、Anthropic、Ollama 等，避免供应商锁
- 企业级特性：工具调用、RAG、评估框架，解决幻觉（hallucination）和安全性问题。

- [Spring AI 官网](https://springdoc.cn/spring-ai/)

## 四、提示词

### 概念

#### 提示词

SpringAI中，我们使用 `Prompt` 类来表示一个提示词，`Prompt` 类中包含一个 `List<Message>` 属性，**用于表示提示词中的消息列表**。

```java
public class Prompt implements ModelRequest<List<Message>> {
    // 消息列表
    private final List<Message> messages;

    private ChatOptions chatOptions;
}
```

#### 消息

用户与大模型中间的对话，通常是由一个或多个消息组成，其中每个消息都需要与一个角色关联；角色表示消息作者的角色。

SpringAI中，我们使用 `Message` 类来表示一个消息，`Message` 类中包含一个 `MessageType` 属性，用于表示消息的类型，通常有四种角色：

- system: 系统角色，用于通过分配特定行为给大模型来创建对话的上下文或范围，通常我们预设的提示词会和 `system` 角色关联
- user: 用户角色，用于表示用户输入的文本，通常我们输入的提问也会和 `user` 角色关联
- assistant: 助手角色，用于表示模型生成的文本，通常大模型生成的答案也会和 `assistant` 角色关联
- tool: 工具角色，用于表示模型调用的函数返回的内容，会和 `tool` 角色关联

```java
public interface Content {

    String getContent();

    Map<String, Object> getMetadata();
}

public interface Message extends Content {
    MessageType getMessageType();
}
```

#### 提示词模板

SpringAI提供了一种模板语法，用于生成提示词，模板语法如下：

```text
You are a helpful AI assistant that helps people find information.
Your name is {name}
You should reply to the user's request with your name and also in the style of a {voice}.
```

模板语法中，`{name}` 和 `{voice}` 是模板变量，模板变量的值会根据调用时传入的参数进行替换

### 实操

#### 提示词

用法

```java

Prompt prompt = new Prompt("xxx");

// 若我们需要创建不同角色的消息，则可以使用
```

```java
@RestController
```

## 五、结构化输出

### BeanOutputConverter

作用：用于将 LLM（大语言模型）返回的非结构化字符串（String）内容，自动解析并转换为 Java 中的结构化对象（POJO/Bean）

用法：

```java
// 创建转换器
BeanOutputConverter<类> beanOutputConverter = new BeanOutputConverter<>(类.class);

// 获取格式化指令并加入到 prompt
String format = beanOutputConverter.getFormat();
PromptTemplate template = new PromptTemplate("帮我返回五个{actor}"主演的电影名{format});
template.create(Map.of("actor",actor,"format",format))
    
// 调用模型并转换结果
String response = chatModel.call(prompt).getResult().getOutput().getText();
类 类名 =  beanOutputConverter.convert(response);
```

例子：

```java
@GetMapping("/chatBeanOutput")
public ActorFilm chatBeanOutput(@RequestParam("actor") String actor) {
    BeanOutputConverter<ActorFilm> beanOutputConverter = new BeanOutputConverter<>(ActorFilm.class);
    String format = beanOutputConverter.getFormat();

    // 1. 在 Prompt 中明确要求
    PromptTemplate promptTemplate = new PromptTemplate(
        "帮我找{actor}主演的五部电影。注意：只输出 JSON 内容，严禁包含任何推理过程、解释或 Markdown 标记。{format}"
    );
    Prompt prompt = promptTemplate.create(Map.of("actor", actor, "format", format));

    // 2. 获取原始响应
    String rawText = openAiChatModel.call(prompt).getResult().getOutput().getText();

    // 3. 执行强制清洗逻辑
    String cleanedText = repairJson(rawText);

    if (cleanedText.isEmpty()) {
        throw new RuntimeException("AI 返回内容为空或无法识别 JSON");
    }

    return beanOutputConverter.convert(cleanedText);
}

/**
 * 强力修复 JSON 字符串：只保留第一个 { 和最后一个 } 之间的内容
*/
private String repairJson(String text) {
    if (text == null || text.isBlank()) return "";

    // 尝试寻找第一个 {
    int start = text.indexOf("{");
    // 尝试寻找最后一个 }
    int end = text.lastIndexOf("}");

    if (start != -1 && end != -1 && start < end) {
        return text.substring(start, end + 1);
    }

    return ""; // 如果连括号都找不着，说明数据彻底坏了
}
```

## 七、工具调用

### 调用流程

- 工具调用流程

1. 在向大模型发起请求时，将工具信息（名称、描述、输入参数）传递给大模型 
2. 大模型根据工具信息，会发送包含工具名称、亲求参数的请求给工具服务
3. 应用程序根据工具名称，识别到对应的工具，调用工具方法
4. 工具执行结果返回给应用程序进行处理（可能是直接返回给用户，也可能是返回给大模型）
5. 应用程序将工具执行结果返回给大模型
6. 大模型利用工具返回的结果，构建返回结果给用户

### Spring AI 实现

两种实现方式：

- 声明式：
  - 实现方法：注解 `@Tool`
  - 适用场景：工具实现完全由开发者掌控
- 编程式：
  - 实现方法：底层的 `MethodToolCallback` `FunctionToolCallback`
  - 适用场景：非控制的工具实现（如三方sdk），或者需要动态获取工具参数

#### 声明式

```java
public class DateTimeTools {

    /**
     * 获取当前时间
     *
     * @return 结果
     */
    @Tool(description = "获取用户的当前日期和时间")
    String getCurrentDateTime() {
        String ans = LocalDateTime.now().atZone(LocaleContextHolder.getTimeZone().toZoneId()).toString();
        System.out.println("进入获取当前时间了：" + ans);
        return ans;
    }

}
```

带参数的工具方法：

```java
@Tool(description = "传入时区，返回对应时区的当前时间给用户")
String getTimeByZoneId(@ToolParam(description = "需要查询时间的时区") ZoneId area) {
    // 根据时区，查询对应的时间
    ZonedDateTime time = LocalDateTime.now().atZone(area);

    // 转换为 2025-07-26 20:00:00 格式的字符串
    // 将输入时区的时间转换为本地时区
    ZonedDateTime localTime = time.withZoneSameInstant(ZoneId.systemDefault());
    DateTimeFormatter formatter = DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss");
    String ans = localTime.format(formatter);
    System.out.println("传入的时区是：" + area + "-" + ans);
    return ans;
}
```
