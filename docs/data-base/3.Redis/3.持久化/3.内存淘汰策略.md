8种：**lru两个、lfu两个、随机两个、根据过期时间淘汰、报错**

1. volatile-lru：当内存不足以容纳新写入数据时，从**设置了过期时间的key**中使用LRU（最近最少使用）算法进行淘汰；
2. allkeys-lru：当内存不足以容纳新写入数据时，从**所有key**中使用LRU（最近最少使用）算法进行淘汰。
3. volatile-lfu：4.0版本新增，当内存不足以容纳新写入数据时，在**过期的key中，使用LFU（最少频率）算法**进行删除key。
4. allkeys-lfu：4.0版本新增，当内存不足以容纳新写入数据时，从**所有key中使用LFU算法**进行淘汰；
5. volatile-random：当内存不足以容纳新写入数据时，从设置了过期时间的key中，随机淘汰数据；。
6. allkeys-random：当内存不足以容纳新写入数据时，从所有key中随机淘汰数据。
7. volatile-ttl：当内存不足以容纳新写入数据时，在设置了过期时间的key中，**根据过期时间进行淘汰**，越早过期的优先被淘汰；
8. noeviction：**默认策略，当内存不足以容纳新写入数据时，新写入操作会报错**。
